# Identity & Keys

The `SB384` class is the root of all identity in os384. Every user, every channel, and every app key is ultimately an `SB384` object — a P-384 elliptic-curve key pair.

---

## SB384

`SB384` is the "basic capability object". It wraps a P-384 key pair and exposes the externally-visible identity formats that flow through the system.

### The "ready template"

Like most lib384 classes, `SB384` follows the **ready template**: the object can be created synchronously, but async initialization (key generation or import) happens in the background. Always `await obj.ready` before accessing key properties:

```typescript
const identity = new SB384()
await identity.ready   // waits for key generation to complete

console.log(identity.userId)        // e.g. "Xk4aB3..."  (43 base62 chars)
console.log(identity.userPublicKey) // "PNk4..." prefix + x,y encoded
```

If you access getters before `ready` resolves, they will throw.

### Constructor

```typescript
new SB384(key?: CryptoKey | JsonWebKey | SBUserPublicKey | SBUserPrivateKey, forcePrivate?: boolean)
```

Accepts several starting points:

- **Nothing** — generates a fresh random P-384 key pair (always private).
- **`CryptoKey`** — imports an existing Web Crypto key.
- **`JsonWebKey`** — imports from JWK format.
- **`SBUserPublicKey`** — reconstructs a public key (cannot sign or decrypt).
- **`SBUserPrivateKey`** — reconstructs a full private key.

`forcePrivate: true` throws if the key cannot provide a private component. Useful to assert that you have signing/decryption capability without adding your own checks.

### Static methods

```typescript
static SB384.newPrivateKey(): Promise<SBUserPrivateKey>
```

Convenience: generates a fresh private key and returns it in wire format. Equivalent to:
```typescript
(await (new SB384()).ready).userPrivateKey
```

### Key identity formats

| Getter | Type | Description |
|---|---|---|
| `userId` | `SB384Hash` | 43-char base62 SHA-384 hash of the public key. Used as channel IDs and user IDs. Cannot be used to reconstruct the key. |
| `hash` | `SB384Hash` | Same as `userId`. |
| `hashB32` | `SB384Hash` | Same hash, base32 encoded. |
| `userPublicKey` | `SBUserPublicKey` | Wire-format public key: x and y coordinates encoded as base62 with a `PNk4` prefix. |
| `userPrivateKey` | `SBUserPrivateKey` | Wire-format private key: x, y, d encoded as base62 with a `Xj34` prefix. |
| `userPrivateKeyDehydrated` | `SBUserPrivateKey` | Compressed private key — only `d` is present, `x` must be derived. |
| `binaryD` | `ArrayBuffer` | Raw private key scalar `d` as an `ArrayBuffer`. |
| `ownerChannelId` | `string` | The channel ID that this key pair "owns" by derivation. If you are the owner of a channel, `ownerChannelId === channelId`. |
| `jwkPublic` | `JsonWebKey` | Public key in JWK format. |
| `jwkPrivate` | `JsonWebKey` | Private key in JWK format. |
| `private` | `boolean` | `true` if this key has a private component. |
| `ySign` | `0 \| 1` | Sign bit of the y-coordinate (used in compressed key encoding). |

### Key encoding reference

os384 keys use a prefix convention for wire-format strings (all base62-encoded):

| Prefix | Meaning |
|---|---|
| `PNk4` | Public key — x and y present |
| `PNk2` | Compressed public key — y is even |
| `PNk3` | Compressed public key — y is odd |
| `Xj34` | Private key — x, y, d present |
| `Xj32` | Compressed private key — x and d present, y even |
| `Xj33` | Compressed private key — x and d present, y odd |
| `XjZx` | Dehydrated private key — only d; x from external source |

## ChannelKeys

`ChannelKeys` extends `SB384` with the additional state needed to talk to a channel server.

```typescript
class ChannelKeys extends SB384
```

It is the base of the `Channel` hierarchy. You will typically not instantiate it directly — use `Channel`, `ChannelSocket`, or `ChannelStream` instead.

### Constructor

```typescript
new ChannelKeys(handleOrKey?: ChannelHandle | SBUserPrivateKey)
```

Can be initialized from a `ChannelHandle` (which may already carry channel server info, channel data, etc.) or directly from a `SBUserPrivateKey`.

### Properties and methods

| Member | Description |
|---|---|
| `channelId` | The channel's public identity (`SB384Hash`). |
| `channelServer` | The URL of the channel server for this instance. Falls back to the global default if not set. |
| `owner` | `true` if the key is the owner key for this channel. |
| `channelData` | The `SBChannelData` structure (owner pub key, channel ID, optional storage token). |
| `handle` | Returns a `ChannelHandle` for this channel. |
| `buildApiBody(path, payload?)` | Constructs a signed `ChannelApiBody` for a given API path. |
| `callApi(path, payload?)` | Makes a signed API call to the channel server. |

## SBEventTarget

`SBEventTarget` is the event emitter base used by `ChannelApi` and `AppMain`. Note: it is entirely **static** — all listeners are global for the class, not per-instance.

```typescript
class SBEventTarget {
  static addEventListener(type, callback, options?)
  static removeEventListener(type, callback, options?)
  static dispatchEvent(event): boolean
  static on(eventName, listener)
  static off(eventName, listener)
  static emit(eventName, ...args)
}
```

For `ChannelApi` event usage (online/offline status events), see [Channel API](./channel#channelapi).

## Type aliases

These type aliases appear throughout the API. They are all `string` at runtime, but carry semantic meaning:

```typescript
type SB384Hash = string        // 43-char base62, SHA-384 hash
type SBUserId  = SB384Hash     // user identity (hash of public key)
type ChannelId = SB384Hash     // channel identity (hash of owner public key)
type SBUserPublicKey  = string // base62 wire format public key  (PNk4...)
type SBUserPrivateKey = string // base62 wire format private key (Xj34...)
```

Guard function:

```typescript
isSBUserId(x: any): x is SBUserId
```
